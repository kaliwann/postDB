Графовые базы в MS SQL Server

Вариант: Почта: Отделения, посылки, почтальоны.

В соответствии с вариантом задания, выполнить следующие действия:

1. Создайте не менее трёх таблиц узлов. Например, одноклассники, сокурсники (одногруппники), учебные дисциплины, книги (жанры книг), фильмы (жанры книг), города.

2. Создайте не менее трёх таблиц рёбер. Например, дружит, проживает, списывает домашнее задание, рекомендует книгу или фильм (жанр). Учитывайте, что графы в SQL Server — направленные. Например, связь "Дружит" обычно двусторонняя, а связь "Платит" — односторонняя. Для таблиц узлов использовать CONNECTION CONSTRAINT, чтобы ограничить узлы, которые может связывать ребро. Таблицы рёбер должны содержать атрибуты самого ребра (например, дата_события, вес_связи или статус).

3. Заполните для каждой таблиц узлов не менее 10 строк.

4. Введите данные в таблицы рёбер для установлению нужной связи между узлами.

5. Используя функцию MATCH, напишите не менее 5 различных запросов к построенной графовой базе данных.  Использовать цепочку из 3-х и более узлов (например: "Найти все фильмы, в которых снимались актеры, работавшие с режиссером X").

6. Используя функцию SHORTEST_PATH, напишите не менее 2 различных запросов к построенной графовой базе данных (используйте как шаблон "+", так и "{1,n}"), например, "Найти кратчайший путь длиной от 1 до 5 шагов с выводом имен всех промежуточных узлов". Требуется использовать функции LAST_NODE и STRING_AGG для отображения пройденного пути.

7. Нарисуйте в https://www.diagrams.net/ диаграммы для пунктов 3-4 (см. видео на YouTube "Графовые базы в MS SQL Server" https://youtu.be/LyXaerp6tRQ). Все диаграммы выполнить в одном файле с расширением .drawio, но каждую диаграмму рисовать на отдельной странице.

8. Визуализируйте граф в Power BI с помощью Force-Directed Graph (см. видео на YouTube "Визуализация графа из MS SQL Server в Power BI с помощью Force-Directed Graph" https://youtu.be/u7DRojqcNnE).

9. Предложите свою задачу на графовую базу данных.
